🛰️ Simulador de GPS

No app real isto é o watchPosition(). Aqui você controla a posição do usuário para testar o geofencing sem sair da cadeira. Arraste o marcador azul no mapa do app, ou use o teleporte:

📍 Posição:
GPS real exige HTTPS (GitHub Pages / túnel) ou localhost. "Trazer para perto" muda o Theatro Municipal para a sua posição, mantendo a geometria — dá para testar andando de verdade.
🕐 Relógio virtual:

Saltos grandes de posição sem avançar o relógio disparam a detecção de "viagem impossível" e o check-in cai na fila de revisão ⚠️ (Admin → Revisão).

📋 Roteiro de testes (10 cenários)
  1. Check-in feliz: teleporte para o Theatro Municipal → botão fica verde → check-in → +30 pts e missão 1/5.
  2. Fora do raio: arraste o marcador ~200 m para longe → botão desabilita mostrando a distância. Use "🧪 testar mesmo assim" para ver o servidor rejeitar com OUT_OF_RANGE.
  3. GPS impreciso: suba a precisão para 150 m → servidor rejeita com LOW_GPS_ACCURACY.
  4. Tolerância: com precisão 50 m, faça check-in um pouco fora do raio — o servidor absorve (raio + min(accuracy, 50)).
  5. Viagem impossível: check-in no Centro → teleporte direto para o Ibirapuera → check-in → concedido mas ⚠️ sinalizado. Avance o relógio +1 h e repita em outro lugar: passa limpo.
  6. Mock location: ligue o toggle → próximo check-in nasce sinalizado.
  7. Completar missão: visite os 5 locais do Centro Histórico → modal de conquista + 200 pts de bônus.
  8. Admin → app: crie um local no Admin, publique, volte ao app → vira card avulso e pin no mapa.
  9. Revogação: Admin → Revisão → revogue um check-in sinalizado → pontos estornados no extrato (Perfil); se uma conquista dependia dele, é revogada junto.
  10. Missão futura: os locais do "Festival de Inverno" estão 🔒. Avance o relógio +1 dia ×21 → eles desbloqueiam.
